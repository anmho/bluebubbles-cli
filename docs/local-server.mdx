---
title: "Local Server"
description: "Manage an installed BlueBubbles Server app on macOS."
---

## Commands

These commands are local process-manager operations, not API calls.

```bash
bluebubbles server start
bluebubbles server status
bluebubbles server stop
bluebubbles server restart
bluebubbles server logs
bluebubbles server logs --source api
```

## App discovery

The CLI checks these default locations:

- `/Applications/BlueBubbles.app`
- `/Applications/BlueBubbles Server.app`
- `~/Applications/BlueBubbles.app`
- `~/Applications/BlueBubbles Server.app`

If your install lives elsewhere:

```bash
bluebubbles config set appPath /Applications/BlueBubbles.app
```

## Launch arguments

Persist launch arguments:

```bash
bluebubbles config set launchArgs --headless,--foreground
```

Or pass them at runtime:

```bash
bluebubbles server start --arg --headless --arg --foreground
```

## Logs

The CLI prefers a configured log file or the log captured from CLI-owned launches.

```bash
bluebubbles server logs --count 100
bluebubbles server logs --follow
bluebubbles config set logPath /path/to/bluebubbles.log
```
